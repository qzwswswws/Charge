# 快速产出导向的横向扩充执行计划

## 1. 文档目的

本文档用于在“论文已经开始写作”的前提下，对后续补充实验进行**重新排程**。  
与 [Horizontal_Expansion_Plan.md](/home/woqiu/下载/git/MI_Algorithm_Workbench/00_AI_Management/Output_Drafts/Horizontal_Expansion_Plan.md) 不同，本文件不再按研究主题排序，而是按：

- **多久能产出第一批可写进论文的结果**
- **实施复杂度多高**
- **能否与现有主线无缝衔接**

进行优先级重排。

一句话原则：

> 先做“今天或明天就能出图、出表、出结论”的内容，再做需要重实现或重预处理的扩展。

---

## 2. 排程原则

### 2.1 排序依据

所有补充项按以下维度综合排序：

1. **出第一批结果所需时间**
2. **对论文直接价值**
3. **是否依赖新代码 / 新数据预处理**
4. **是否会打乱当前主线**

### 2.2 时间定义

本文中的耗时采用两种口径：

- `首批结果耗时`：多久能拿到第一批可判断方向的结果
- `完整收尾耗时`：多久能形成论文可直接引用的完整表格或图

### 2.3 当前主线不变

无论补什么，当前论文主线仍然保持：

- `22导 / 4分类` baseline
- `22 -> 8 -> 3 -> 2` 四分类退化
- `2导 / 2分类` 工程落地
- `KD` 作为补充性补偿实验

也就是说，下面所有扩充都属于“补强”和“横向展开”，不是重开主线。

---

## 3. 总体排期概览

## 第一梯队：最快出结果，优先立刻做

这些项目几乎都能在 `0.5` 到 `1` 天内看到第一批可用结果：

1. 复杂度与模型大小分析
2. confusion matrix 与类间错误分析
3. KD 从 `5` 被试扩到 `9` 被试
4. 通道组合 pilot

## 第二梯队：1 到 3 天内出主结果

这些项目需要一定批量运行，但回报很高：

5. 多随机种子复现与统计检验
6. 训练数据比例敏感性实验

## 第三梯队：3 到 7 天内出结果

这些项目对论文很有帮助，但需要更多实现或预处理工作：

7. 与传统算法对比实验
8. `BCI Competition IV 2b` 外部验证

## P2. confusion matrix 与类间错误分析

### 排名原因

这项工作同样几乎不需要新增训练，属于“立刻可以把现有数字讲得更透”的典型项目。

### 建议输出

- `22导 / 4分类` confusion matrix
- `2导 / 4分类` confusion matrix
- `2导 / 2分类` confusion matrix
- 每类召回率 / 精确率
- 一段错误模式解读

### 重点问题

重点不是简单画图，而是回答：

- 双导条件下最先崩的是不是 `脚` 和 `舌头`
- 左右手在双导条件下是否仍保持较好分离
- 这是否与中央区生理信息分布相一致

### 预估耗时

- 首批结果耗时：`2~6 小时`
- 完整收尾耗时：`0.5~1 天`

### 论文价值

它能显著提升：

- 讨论部分质量
- 退化原因的可解释性

### 当前优先级

`最高`

## P5. 多随机种子复现与统计检验

### 排名原因

这项工作是论文严谨性提升最大的项目之一，但相比前四项，它需要更多训练时间，因此放在第二梯队。

### 推荐先做的对象

先不要全铺开，先做最关键的三条：

- `22导 / 4分类 baseline`
- `2导 / 4分类 baseline`
- `2导 / 2分类 baseline`

若资源允许，再加：

- `KD student / 2导 / 4分类`

### 推荐种子数

- 最少 `3` 个种子
- 理想 `5` 个种子

### 建议统计输出

- `mean ± std`
- paired t-test 或 Wilcoxon
- 效应方向说明

### 推荐执行策略

先做 `pilot 被试 × 多 seed`，确认方差量级；  
若方差可控，再扩到 `9` 被试。

### 预估耗时

- 首批结果耗时：`1 天`
- 完整收尾耗时：`2~3 天`

### 论文价值

非常高。  
它会让整篇论文从“结果成立”变成“结果更可信”。

### 当前优先级

`高`

## 5. 推荐并行方式

如果希望尽快形成一批可以写进论文的新材料，建议按下面方式并行：

## 线路 A：当天能出结果

- P1 复杂度分析
- P2 confusion matrix / 错误分析

这两项几乎不依赖新训练，最适合立刻启动。

## 线路 B：现成脚本快速扩展

- P3 KD 扩到 `9` 被试
- P4 通道组合 pilot

这两项代码基础已经比较完整，属于“最容易快速出新表”的扩展。

## 线路 C：中短期高价值

- P5 多随机种子复现
- P6 训练数据比例敏感性

这两项更适合在前两条线启动后同步排队执行。

## 线路 D：较慢但完整度高

- P7 传统算法对照
- P8 `2b` 外部验证

这两项最适合作为第二阶段工程。

## 6. 建议的一周节奏

如果按“尽快拿到可写作的新结果”来安排，我建议是：

### 第 1 天

- 完成 P1 复杂度分析
- 完成 P2 confusion matrix / 错误分析
- 启动 P3 KD 全被试扩展

### 第 2 天

- 完成 P3 汇总
- 启动 P4 通道组合 pilot
- 搭好 P5 多 seed 运行器

### 第 3 天

- 完成 P4 汇总
- 跑 P5 第一轮 pilot seeds
- 启动 P6 数据比例 pilot

### 第 4~5 天

- 完成 P5 / P6
- 判断是否有必要扩到全 `9` 被试

### 第 6~7 天

- 视前面结果，决定是先做 P7 传统对照
- 还是先做 P8 `2b` 外部验证

---

## 7. 如果只能立刻启动四项

如果你希望现在就明确“最先开跑的四项”，我的推荐是：

1. **P1 复杂度与模型大小分析**
2. **P2 confusion matrix 与类间错误分析**
3. **P3 KD 从 5 被试扩到 9 被试**
4. **P4 通道组合 pilot**

原因很简单：

- 它们最容易尽快给论文带来新的图、表、解释段落
- 不会明显打乱现有主线
- 代码与数据准备成本最低

---

## 8. 一句话结论

接下来最优的推进方式不是“按研究主题平均发力”，而是：

> 先用复杂度分析、错误分析、KD 扩被试和通道组合 pilot 在最短时间内拿到一批新的可写结果，再进入多随机种子复现、数据比例实验、传统对照和 `2b` 外部验证这些更耗时但价值更高的补充。
